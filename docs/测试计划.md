# 测试计划（草案）

> 负责人：hjh　日期：6/18（初稿）→ 6/20（更新）　随开发推进持续补充，最终用于课程报告"测试结果与分析"章节

## 1. 测试目标

验证校园二手交易平台各模块功能正确、数据库完整性约束有效、关键业务流程符合预期。

## 2. 测试类型

| 类型 | 说明 | 负责    |
|---|---|-------|
| 完整性约束测试 | 验证主键/唯一/外键/CHECK/非空约束生效（见 `sql/test/integrity_test.sql`） | hjh   |
| 功能测试 | 各接口正常/异常输入的返回是否正确 | 各模块成员 |
| 业务流程测试 | 交易/社交/治理三条主流程端到端验证 | 全员    |

## 3. 完整性约束测试用例（核心表，6/18 草案 → 6/20 与 SQL 对齐）

| 编号 | 用例 | 操作 | 预期结果 |
|---|---|---|---|
| IC-01 | 用户名唯一 | 插入重复 username | 触发 uk_username，报错 |
| IC-02 | 商品价格非负 | 插入 price = -1 | 触发 chk_prod_price，报错 |
| IC-03 | 卖家外键 | 插入不存在的 seller_id | 触发 fk_prod_seller，报错 |
| IC-04 | 分类外键 | 插入不存在的 category_id | 触发 fk_prod_category，报错 |
| IC-05 | 地址外键 | 插入不存在的 user_id（address 表） | 触发 fk_addr_user，报错 |
| IC-06 | 标题非空 | title = NULL | 报错，拒绝插入 |

> 以上 6 条用例 SQL 已写入 `sql/test/integrity_test.sql`（默认注释，测试时逐条取消注释执行）。

## 4. 商品模块功能测试用例（6/19 补充）

测试数据已灌入 `sql/test/more_products.sql`（15 条），配合 `init_data.sql`（4 条）共 19 条在售商品。

| 编号 | 用例 | 验证方式 | 预期结果 |
|---|---|---|---|
| F-01 | 在售商品总数 | `SELECT COUNT(*) FROM product WHERE status=1` | 19 条 |
| F-02 | 关键词搜索"教材" | 标题 LIKE 模糊匹配 | 返回含"教材"的在售商品 |
| F-03 | 按分类筛选（category_id=22） | `WHERE category_id=22` | 仅返回"电脑配件"分类商品 |
| F-04 | 分页第1页（每页12条） | `LIMIT 0,12` 按时间倒序 | ≤12 条，时间倒序 |
| F-05 | 分页第2页（偏移12） | `LIMIT 12,12` 按时间倒序 | 返回剩余记录 |
| F-06 | 商品详情联表查询 | LEFT JOIN category + user | 含分类名、卖家昵称 |

> SQL 脚本见 `sql/test/product_test.sql`（6 条功能查询 + 4 条约束验证，后者默认注释）。
