## 5. 功能测试用例

### 5.1 商品模块功能查询（6/19 建立）

测试数据已灌入 `sql/test/more_products.sql`（15 条），配合 `init_data.sql`（4 条）共 19 条在售商品。

| 编号 | 用例 | 验证方式 | 预期结果 |
|---|---|---|---|
| F-01 | 在售商品总数 | `SELECT COUNT(*) FROM product WHERE status=1` | 19 条 |
| F-02 | 关键词搜索"教材" | 标题 LIKE 模糊匹配 | 返回含"教材"的在售商品 |
| F-03 | 按分类筛选（category_id=22） | `WHERE category_id=22` | 仅返回"电脑配件"分类商品 |
| F-04 | 分页第1页（每页12条） | `LIMIT 0,12` 按时间倒序 | ≤12 条，时间倒序 |
| F-05 | 分页第2页（偏移12） | `LIMIT 12,12` 按时间倒序 | 返回剩余记录 |
| F-06 | 商品详情联表查询 | LEFT JOIN category + user | 含分类名、卖家昵称 |

> SQL 脚本见 `sql/test/product_test.sql`（6 条功能查询 + 4 条约束验证）。

### 5.2 后台统计功能（6/21 建立）

| 编号 | 用例 | 验证内容 |
|---|---|---|
| S-01 | 用户总数 | `countUsers()` 返回与 `SELECT COUNT(*) FROM user` 一致 |
| S-02 | 商品总数 | `countProducts()` 返回所有商品数（含各状态） |
| S-03 | 订单总数 | `countOrders()` 返回所有订单数 |
| S-04 | 成交总额 | `totalSales()` 仅统计 status=3 的已完成订单 |
| S-05 | 热门分类 | `hotCategories()` 按在售商品数倒序 |
| S-06 | 活跃用户 Top5 | `activeUsers()` 按下单数倒序，返回前 5 名 |
