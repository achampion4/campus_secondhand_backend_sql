# 测试计划（草案）

> 负责人：hjh　日期：6/18　（随开发推进持续补充，最终用于课程报告"测试结果与分析"章节）

## 1. 测试目标

验证校园二手交易平台各模块功能正确、数据库完整性约束有效、关键业务流程符合预期。

## 2. 测试类型

| 类型 | 说明 | 负责 |
|---|---|---|
| 完整性约束测试 | 验证主键/唯一/外键/CHECK/非空约束生效（见 `sql/test/integrity_test.sql`） | 同学E |
| 功能测试 | 各接口正常/异常输入的返回是否正确 | 各模块成员 |
| 业务流程测试 | 交易/社交/治理三条主流程端到端验证 | 全员 |

## 3. 完整性约束测试用例（核心表，6/18 草案）

| 编号 | 用例 | 操作 | 预期结果 |
|---|---|---|---|
| IC-01 | 用户名唯一 | 插入重复 username | 报错，拒绝插入 |
| IC-02 | 商品价格非负 | 插入 price = -1 | 触发 chk_prod_price，报错 |
| IC-03 | 卖家外键 | 插入不存在的 seller_id | 触发外键约束，报错 |
| IC-04 | 分类外键 | 插入不存在的 category_id | 触发外键约束，报错 |
| IC-05 | 标题非空 | title = NULL | 报错，拒绝插入 |
