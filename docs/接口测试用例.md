# 接口测试用例（用户模块 + 商品模块）

> 负责人：霍俊豪　日期：6/20　用于功能验证与课程报告"测试结果与分析"章节
> 测试方式：Postman / Apifox 或前端页面操作；带 🔒 的接口需在请求头加 `Authorization: Bearer <token>`

## 一、用户模块

| 编号 | 接口 | 方法 | 输入 | 预期结果 |
|---|---|---|---|---|
| U-01 | /api/user/register | POST | 合法用户名/密码/昵称 | code=200，返回新用户ID |
| U-02 | /api/user/register | POST | 已存在的用户名 | code=500，提示"用户名已存在" |
| U-03 | /api/user/register | POST | 密码少于6位 | code=400，参数校验失败 |
| U-04 | /api/user/login | POST | 正确用户名+密码 | code=200，返回 token 与 user |
| U-05 | /api/user/login | POST | 错误密码 | code=500，提示"密码错误" |
| U-06 | /api/user/login | POST | 不存在的用户名 | code=500，提示"用户不存在" |
| U-07 | 🔒 /api/user/profile | GET | 携带有效 token | code=200，返回个人信息(无密码) |
| U-08 | 🔒 /api/user/profile | GET | 不带/错误 token | code=401，未登录 |
| U-09 | 🔒 /api/user/profile | PUT | 修改昵称 | code=200，再查询昵称已更新 |
| U-10 | 🔒 /api/address (POST/PUT/DELETE) | — | 增/改/删地址 | code=200，列表相应变化 |
| U-11 | 🔒 /api/address | PUT | 修改他人地址 | code=500，提示"无权操作该地址" |

## 二、商品模块

| 编号 | 接口 | 方法 | 输入 | 预期结果 |
|---|---|---|---|---|
| P-01 | /api/category/list | GET | — | code=200，返回分类列表 |
| P-02 | 🔒 /api/product | POST | 合法商品表单 | code=200，返回商品ID，状态为在售 |
| P-03 | 🔒 /api/product | POST | 缺少标题/分类 | code=400，参数校验失败 |
| P-04 | /api/product/list | GET | 无参数 | code=200，返回分页(默认每页12) |
| P-05 | /api/product/list | GET | keyword=教材 | 仅返回标题含"教材"的在售商品 |
| P-06 | /api/product/list | GET | categoryId=22 | 仅返回该分类在售商品 |
| P-07 | /api/product/list | GET | page=2 | 返回第2页数据，total 不变 |
| P-08 | /api/product/detail/{id} | GET | 存在的商品ID | code=200，返回详情+图片，浏览量+1 |
| P-09 | /api/product/detail/{id} | GET | 不存在的ID | code=500，提示"商品不存在" |
| P-10 | 🔒 /api/product | PUT | 编辑他人商品 | code=500，提示"无权编辑他人商品" |
| P-11 | 🔒 /api/product/offShelf/{id} | PUT | 下架自己的商品 | code=200，列表中不再出现该商品 |
| P-12 | 🔒 /api/file/upload | POST | 上传图片文件 | code=200，返回 /uploads/xxx 路径 |
