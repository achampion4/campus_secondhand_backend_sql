# 校园二手交易平台 —— 服务器部署操作手册

> 负责人：同学E（部署/运行环境）
> 适用环境：阿里云 ECS（Ubuntu 22.04），服务器已运行其它项目，本项目使用**独立端口**与之共存。
> 技术栈：前端 Vue3（打包静态）+ 后端 Spring Boot（jar）+ MySQL + Nginx。

---

## 一、部署架构与端口规划

```
浏览器 → Nginx(:8081 前端静态 + 反向代理)
                 ├── /api/**     → Spring Boot (:8090)
                 └── /uploads/** → Spring Boot (:8090)
                                      └── MySQL (:3306, 库 campus_secondhand)
```

| 组件 | 端口 | 对外 | 说明 |
|---|---|---|---|
| Nginx（前端） | **8081** | 是（安全组放行） | 用户访问入口 |
| Spring Boot（后端） | **8090** | 否（仅本机） | 接口服务 |
| MySQL | 3306 | 否 | 数据库 |

> 选独立端口是为了不影响服务器上已有的项目。前端代码用相对路径 `/api`，**不写死 IP/端口**，换端口只改 Nginx 与后端配置，前端不用动。

## 二、环境准备（首次部署）

SSH 登录服务器后安装基础软件：
```bash
ssh root@你的公网IP
apt update
apt install -y openjdk-17-jdk mysql-server nginx
java -version          # 确认 17
```
> MySQL 安装过程中可能提示设置 root 密码，记住这个密码，后面 `application.yml` 和数据库导入都会用到。

阿里云控制台 → ECS 安全组 → 入方向放行 **8081**（22/80 一般已放行；8090、3306 不要对公网开放）。

## 三、数据库部署

### 1. 设置 MySQL 密码
```bash
mysql
```
```sql
ALTER USER 'root'@'localhost' IDENTIFIED WITH mysql_native_password BY '你的强密码';
FLUSH PRIVILEGES;
EXIT;
```

### 2. 上传并按顺序导入建表脚本
本地终端把 sql 整个目录传上去（**注意用 -r 递归**，test 子目录里的测试数据也要传）：
```bash
scp -r sql/ root@你的公网IP:/root/
```
服务器上**严格按顺序**执行（顺序错会因外键依赖报错）：
```bash
cd /root/sql
mysql -uroot -p < schema.sql               # 核心5表 + 建库(会DROP重建)
mysql -uroot -p < order_tables.sql         # 订单表
mysql -uroot -p < social_tables.sql        # 收藏/消息/评价/举报表
mysql -uroot -p < init_data.sql            # 初始数据(账号/分类/商品)
mysql -uroot -p < test/more_products.sql   # 测试商品(可选，注意在test子目录)
```

### 3. 验证
```sql
USE campus_secondhand;
SHOW TABLES;     -- 应为 11 张表
```
11 张：user、address、category、product、product_image、order_info、order_item、favorite、message、review、report。

## 四、后端部署

### 1. 本地打包
后端项目目录，先确认 `application.yml`：
- `spring.datasource.password` 改成服务器 MySQL 密码
- `spring.datasource.url` 用 `localhost:3306`
```bash
mvn clean package -DskipTests
# 产物：target/secondhand-1.0.0.jar
```

### 2. 上传并运行（端口 8090 + DeepSeek Key）
```bash
# 先创建目标目录
ssh root@你的公网IP "mkdir -p /root/app"
scp target/secondhand-1.0.0.jar root@你的公网IP:/root/app/
```
服务器上启动（用启动参数覆盖端口与 AI Key，避免把 key 写进 jar）：
```bash
cd /root/app
nohup java -jar secondhand-1.0.0.jar \
  --server.port=8090 \
  --deepseek.api-key=sk-你的DeepSeekKey \
  > app.log 2>&1 &
```
### 3. 验证后端
```bash
tail -f app.log                       # 看到“启动成功”，Ctrl+C 退出
curl http://localhost:8090/api/ping   # 返回 json 即 OK
```

> AI 导购功能日志：调用时 `app.log` 会打印 `[AI导购] ...`（需求、商品数、DeepSeek 返回、耗时）。
> DeepSeek Key 到 platform.deepseek.com 注册获取。

## 六、Nginx 配置

服务器新建独立配置 `/etc/nginx/conf.d/campus.conf`（**不要动现有项目的配置**）：
```nginx
server {
    listen 8081;
    server_name _;

    root /var/www/campus;
    index index.html;

    # Vue history 路由：刷新不 404
    location / {
        try_files $uri $uri/ /index.html;
    }
    # 接口转发到后端
    location /api/ {
        proxy_pass http://127.0.0.1:8090;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
    }
    # 上传图片访问转发到后端
    location /uploads/ {
        proxy_pass http://127.0.0.1:8090;
    }
}
```
检查并重载：
```bash
nginx -t && systemctl reload nginx
```

## 八、常见问题排查

| 现象 | 原因 / 处理 |
|---|---|
| 网页打不开 | 安全组没放行 8081 / Nginx 没起（`systemctl status nginx`） |
| 接口 502、登录转圈 | 后端没起或挂了，看 `tail -50 /root/app/app.log`；确认 8090 在跑（`ss -tlnp \| grep 8090`） |
| 刷新页面 404 | Nginx 缺 `try_files ... /index.html` |
| 表不存在报错（Table doesn't exist） | 建表脚本没按顺序跑全（schema→order→social），或漏了某张表 |
| 下单提示"商品不存在" | 没导入 `test/more_products.sql` 或商品数不够 |
| 图片裂图（详情页不显示） | 初始数据是占位假图（`/images/sample1.jpg`）属正常；真实上传的图片走 `/uploads/` 正常显示 |
| AI 提示"服务不可用" | DeepSeek Key 没填 / 错了 / 欠费，看 `app.log` 中 `[AI导购] 调用失败` 具体报错 |
| 后端起不来 | 多为数据库密码/连接不对，看 `tail -30 /root/app/app.log` 末尾异常 |
| 部署后前端页面是旧的 | 清浏览器缓存（Ctrl+Shift+Delete）或硬刷新（Ctrl+F5） |
| Nginx 配置报错 | `nginx -t` 查看具体错误行，常见：少分号、路径不存在 |

## 九、更新 / 重启速查

**更新后端**（改了 .java/.xml/.yml 后）：
```bash
# 本地重新打包 → scp 上传新 jar
mvn clean package -DskipTests
scp target/secondhand-1.0.0.jar root@你的公网IP:/root/app/

# 服务器上停旧进程、启新进程
ssh root@你的公网IP
pkill -f secondhand-1.0.0.jar        # 停旧进程
sleep 2                               # 等旧进程完全退出
cd /root/app && nohup java -jar secondhand-1.0.0.jar \
  --server.port=8090 \
  --deepseek.api-key=sk-你的Key \
  > app.log 2>&1 &
# 等几秒后验证
tail -5 app.log                       # 看到"启动成功"即可
```

**更新前端**：
```bash
npm run build           # 本地打包
scp -r dist/* root@你的公网IP:/var/www/campus/   # 覆盖
# Nginx 不用重启，刷新浏览器即可（必要时 Ctrl+F5 硬刷新）
```

**仅重启后端**（没改代码，进程挂了）：
```bash
ssh root@你的公网IP
pkill -f secondhand-1.0.0.jar || true   # 可能在跑也可能已挂
cd /root/app && nohup java -jar secondhand-1.0.0.jar \
  --server.port=8090 \
  --deepseek.api-key=sk-你的Key \
  > app.log 2>&1 &
```
